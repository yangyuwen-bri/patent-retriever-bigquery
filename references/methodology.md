# Patent-Pro 方法论（精简版）

## 检索做到极致

1. 单源约束：仅使用 Google BigQuery 专利表（字段可审计）。
2. 两段式策略：`concept_scan -> query_plan`，再执行多轮过滤检索。
3. 样本门槛：核心样本>=20；不足 20 自动扩检轮。
4. 证据映射：每条关键结论绑定 `publication_number`。
5. 去重策略：按 `publication_number` 去重，保留最新文献。

## 创新挖掘做到极致

1. 使用“要素-机制-效果-场景”矩阵发现空白点。
2. 每个候选方向必须给出：
- 差异特征
- 技术效果
- 实施路径
- 风险与规避
- 证据支撑
3. 输出可申请性评分：新颖性、创造性、工程可行性。

## 交底书做到专业

1. 先权利要求树，再说明书正文，再附图映射。
2. 附图必须与正文引用一致。
3. 无实验数据时，不输出定量对比实验图。
4. 所有术语保持一致，不得前后漂移。

## 质量门禁

1. `concept_scan/query_plan/retriever/innovation` 任一 schema 不通过则 fail。
2. 检索不足、证据不足、图文不一致，均不得通过。
3. 输出必须可追溯：run_id、文件路径、证据来源。
